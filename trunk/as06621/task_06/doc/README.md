# Цель работы

Ознакомление с открытыми программными проектами экосистемы PLCnext, а также получение практических навыков участия в их развитии.

# Работа с открытыми проектами

1. Провести анализ следующих открытых проектов:

* документация по работе с технологией [PLCnext](https://www.plcnext-community.net/en/) — [PLCnext-howto](https://github.com/savushkin-r-d/PLCnext-howto);

* управляющее программное обеспечение для контроллеров [PLCnext Control](https://www.phoenixcontact.com/online/portal/de?1dmy&urile=wcm%3apath%3a/dede/web/main/products/subcategory_pages/PLCnext_Controls_P-21-14/30b12f75-d769-4f0e-a783-4986ae3ae247) — [ptusa-main](https://github.com/savushkin-r-d/ptusa_main);

* расширение для [EPLAN Electric P8](https://www.eplan-russia.ru/reshenija-eplan/platforma-eplan/eplan-electric-p8/) — [EPLANner](https://github.com/savushkin-r-d/EasyEPLANner);

* демонстрационный проект [T1-PLCnext-Demo](https://github.com/savushkin-r-d/T1-PLCnext-Demo).

2. Принять участие в развитии проектов путём:

* оформления замечаний и предложений в виде `issue`;
* создания запросов на внесение изменений (`pull request`);
* предложения исправлений в документации и исходном коде;
* разработки идей по расширению функциональности.

# Ход выполнения работы

## Изучение открытых проектов PLCnext

В рамках выполнения лабораторной работы были рассмотрены несколько открытых репозиториев, применяемых при разработке и эксплуатации систем автоматизации на базе PLCnext.

### Документация PLCnext-howto

Была проанализирована документация PLCnext-howto, содержащая описание ключевых принципов функционирования технологии PLCnext. В процессе изучения рассмотрены:

* архитектурные особенности PLCnext;
* порядок создания, запуска и настройки проектов;
* способы взаимодействия контроллера с SCADA-системами;
* структура системных и пользовательских Lua-скриптов.

В ходе работы были выявлены отдельные неточности формулировок и орфографические ошибки.

### Управляющая программа ptusa-main

Затем был изучен проект ptusa-main, являющийся базой для разработки управляющих программ на контроллерах PLCnext. В процессе анализа рассмотрены:

* механизм запуска приложения и инициализации среды выполнения;
* загрузка Lua-скриптов и контроль их версий;
* функционирование менеджеров ввода-вывода, устройств и технологических объектов;
* использование режима отладки и вывод диагностической информации.

Полученные сведения позволили глубже понять логику работы управляющего приложения.

### Дополнение для EPLAN Electric P8 — EPLANner

Также было изучено дополнение EPLANner, предназначенное для интеграции с EPLAN Electric P8. В ходе работы определено:

* назначение и область применения дополнения;
* перечень данных, формируемых на основе электрических схем EPLAN;
* способы использования полученных данных в проектах PLCnext.

Дополнение существенно упрощает перенос информации из схемотехнической части в программную реализацию.

### Демо-проект T1-PLCnext-Demo

Заключительным этапом стало изучение демонстрационного проекта T1-PLCnext-Demo. В рамках анализа:

* рассмотрена структура каталогов и состав проекта;
* изучены системные и пользовательские Lua-скрипты;
* проанализирована конфигурация модулей ввода-вывода;
* выполнен запуск проекта в режиме отладки.

Проект был успешно запущен без ошибок, что подтвердило корректность его конфигурации и работоспособность.

# Заключение

В результате выполнения лабораторной работы были обнаружены неточности в документации и выявлены аспекты, которые могут быть улучшены в исходном коде и структуре проектов. Полученные знания позволяют эффективно использовать контроллер AXC F 2152 для разработки и отладки управляющих программ, а также уверенно работать с открытыми репозиториями и инструментами экосистемы PLCnext.
