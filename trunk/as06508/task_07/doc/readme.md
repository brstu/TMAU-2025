1 Изучить следующий открытый проект - an open source asynchronous framework with a rich set of abstractions for fast and comfortable creation of C++ microservices, services and utilities - userver.

2 Внести свой вклад в проект (в виде оформления вопросов (issue) и запросов на внесение изменений (pull request)):

предложить исправления в ошибках в документации (синтаксические, орфографические и т.п.);
предложить исправления в исходном коде;
предложить новую функциональность.

В рамках лабораторной работы был изучен проект userver — высокопроизводительный асинхронный C++ фреймворк для создания микросервисов.

Цель работы — внести вклад в проект в виде issues

## Issue 1 — Устаревший пример в документации (метод Execute)

В документации показан пример использования PostgreSQL API, в котором применяется метод:

```
Execute()
```

Однако в актуальной версии userver метод помечен как deprecated.
Пример вводит пользователей в заблуждение.

Было создано issue с предложением заменить метод на актуальный API (ExecuteOptional), а пример — обновить.

## Issue 2 — Ошибка в README (отсутствует include <string>)

В примере из README используется:

```
As<std::string>()
```

Однако в списке include файлов нет:

```
#include <string>
```

Что приводит к ошибке компиляции при копировании примера.
Создано issue с предложением исправить пример.
