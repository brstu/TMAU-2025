Работа с открытыми проектами

В ходе выполнения лабораторной работы были изучены два открытых проекта:

T1-PLCnext-Demo

ptusa_main

В процессе работы обнаружены ошибки в документации, влияющие на корректность сборки и работы ПО. По каждой из них были созданы issue.

###Issue 1 — T1-PLCnext-Demo

Тема: Ошибки в инструкции по настройке launch.vs.json для Visual Studio

В руководстве по работе с проектом T1-PLCnext-Demo была обнаружена ошибка в описании файла launch.vs.json. Инструкция предлагает использовать неправильные ключи и некорректный формат аргументов.

Обнаруженные проблемы:

Использование cwd вместо currentDir — из-за этого рабочая директория процесса не устанавливается.

Аргументы передаются без --, поэтому ptusa_main игнорирует переданные пути и ищет файлы Lua в неверной директории.

В результате отладка невозможна: возникает ошибка sys.io.lua not found.

Выполненные действия:

Создан issue с подробным разбором проблемы.

Предложено корректное рабочее содержимое launch.vs.json.

###Issue 2 — ptusa_main

Тема: Некорректное указание версий Toolchain и SDK

В документации проекта указано:

Toolchain — 2023.0 LTS

SDK — 2022.0.4 LTS

Это логически неверно: Toolchain и SDK должны быть одной серии версий. Разные версии несовместимы.

Выполненные действия:

Создан issue, в котором указано на ошибку.

Заметил, что создание issue в этом проекте отличается от предыдущего. Здесь необходимо обязательно указать вид Issue (в моем случае Documentation Issue),  Issue Type (в моем случае Incorrect Information). Нужно также указать Documentation Location. Я указал:

```
https://github.com/savushkin-r-d/ptusa_main/blob/master/docs/russian_readme/readme.md
Раздел 5:[Как запустить проект]
176 строка
```

Также заполнил поля Issue Description и Issue Improvement. Ниже установил язык: Both и Target Audience: End Users

Предложена корректная формулировка о необходимости совпадения версий.
