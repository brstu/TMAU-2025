# Работа с открытыми проектами  #
## - Документация работы с технологией [PLCnext](https://www.plcnext-community.net/en/) - [PLCnext-howto](https://github.com/savushkin-r-d/PLCnext-howto); ##

В данном проекте в [описании на русском языке](https://github.com/savushkin-r-d/PLCnext-howto/blob/master/HowTo%20build%20program%20Hello%20PLCnext/readmeRU.md) при описании развертывания проекта под заголовком **Использование MS Visual Code** допущена ошибка. Правильное название среды разработки **MS Visual Studio Code**.
В связис этим предлагается изменить название на правильное, так как данное несоответствие названия может ввести в заблуждение из-за наличия похожей по названию среды разработки MS Visual Studio. 
Созданы *issue* по данной проблеме. Также в проект добавлен *pull request*, где данная проблема решена.
## - Управляющая программа для контроллеров [PLCnext Control](https://www.phoenixcontact.com/online/portal/de?1dmy&urile=wcm%3apath%3a/dede/web/main/products/subcategory_pages/PLCnext_Controls_P-21-14/30b12f75-d769-4f0e-a783-4986ae3ae247) - [ptusa-main](https://github.com/savushkin-r-d/ptusa_main);##

В [документации](https://github.com/savushkin-r-d/ptusa_main/blob/master/docs/russian_readme/readme.md) проекта содержиться название файла **CMakeList.txt**. При этом обычно для CMake файлов используется название **CMakeLists.txt**. Файл с таким названием и находиться в проекте. 
Создан *pull request*, который меняет данное название на корректное.

#### **Вывод:** работа с открытыми проектами позволяет разработчику получать новые навыки и подходы к работе. Также это помогает лучше понимать чужой код, правильно работать с документацией. Впоследствии все это можно применять в собственных проектах, а также ориентироваться на частые ошибки с целью их недопущения.####
