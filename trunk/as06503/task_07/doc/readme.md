## 2
### README.md
При проверке файла было обнаружено, что изображение с текстом **upastebin Docker build**
в верхней части файла ведет к пустой странице. В markdown коде данного файла было обнаружено, 
что ссылка ведет к странице Workflow раздела Github Actions.

Ссылка имела следующее содержание: ```https://github.com/userver-framework/upastebin/actions/workflows/docker.ymal```.
Из-за неправильно указанного расширения файла не мог осуществитьтся корректный переход по ссылке, при этом если изменить расширение на правильное:
``` https://github.com/userver-framework/upastebin/actions/workflows/docker.yaml```, происходит корректное отображение страницы.
Предложен PR, для решения данной ошибки документации.

Также, при просмотре остальных ссылок, обнаружено, что файлы workflow имеют расширение ```.yml``` и ```.yaml```.
Исторически .yml появилось из-за ограничений старых файловых систем (расширение 3 символа).
Обычно выбирают .yml, потому что короче.
Но в некоторых проектах предпочитают .yaml ради читаемости.
При этом в одном проекте лучше не смешивать оба варианта, а придерживаться одного стандарта.
Составлен issue с рекомендацией унифицировать файлы (например до 3-символьного варианта).


**Вывод:** при работе с открытым проектом userver были обнаружены ошибки, касающиеся yaml-файлов и ссылок к ним.
Предложены варианты исправления в исходном репозитории.
