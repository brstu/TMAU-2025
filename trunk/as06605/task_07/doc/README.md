# Работа с открытым проектом userver

## Изучение проекта
- Найти официальный репозиторий **userver** на GitHub.  
- Ознакомиться с:
  - README и документацией (overview, примеры использования).
  - Структурой исходного кода (`core`, `tests`, `samples`).
  - Системой сборки (CMake, зависимости).

## Анализ документации
- Проверить README, Wiki, комментарии в коде на наличие:
  - Орфографических ошибок.
  - Неточностей в синтаксисе примеров.
  - Устаревших ссылок.
- Составить список найденных ошибок и предложить исправления.
