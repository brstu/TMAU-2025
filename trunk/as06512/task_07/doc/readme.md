## Ход выполнения

__Изучение открытого проекта__

В ходе выполнения лабораторной работы изучил предложенный открытый проект. Вкратце: 

userver — это открытый фреймворк на C++, предназначенный для быстрой и удобной разработки микросервисов и высоконагруженных сервисов.

Его основная идея — позволить разработчикам писать простой, линейный код, как в синхронных приложениях, но при этом получить все преимущества асинхронной работы и высокую производительность. Это достигается за счёт использования корутин, которые позволяют "приостанавливать" выполнение функции на время ожидания ответа от базы данных или другого сервиса, не блокируя при этом весь поток. Таким образом, один поток может эффективно обрабатывать тысячи одновременных подключений.

Фреймворк предоставляет готовые и хорошо интегрированные решения для типичных задач: подключение к различным базам данных, работа с HTTP, gRPC, кеширование, логирование, сбор метрик и динамическое изменение конфигурации без перезапуска сервиса.

Изначально userver был разработан внутри компании Яндекс для своих сервисов. Сейчас проект развивается как open source под лицензией Apache 2.0 и используется другими компаниями.  

__Внесение своего вклада в проект в виде оформления вопросов__

В ходе исследования документации и кодовой базы были выявлены направления для совершенствования проекта. План участия в развитии проекта включал:

- Создание отчётов (issue) по найденным в документации неточностям, таким как опечатки и возможности для улучшения ясности формулировок.
- Разработку рекомендаций по оптимизации определённых фрагментов исходного кода.
- Предложение концепций для реализации нового функционала или развития существующих возможностей.

Вся работа велась в соответствии с практиками взаимодействия с открытыми проектами и внутренними стандартами фреймворка userver.
